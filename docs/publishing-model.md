# Publishing Model

## Repository Roles

This repository is private and authoring-focused. Students should receive materials from a separate read-only repository.

## Publishing Principle

- Assignments may be published before class.
- Solutions may only be published after the corresponding lecture is completed.
- Unreleased solutions must never appear in student-accessible git history.

## Minimal Publish Contract

The future export flow will use:

- normalized content under `lectures/`
- release state stored in `publish/lectures.yaml`

The export implementation is intentionally deferred until after legacy materials are migrated and normalized.
